# Feature delta benchmark

- **Scene:** Cornell box
- **Resolution:** 640x360
- **Samples (baseline):** 4
- **Timed reps (median):** 5, warmup: 2
- **OpenMP threads:** 2

| Configuration | Median time (s) | vs baseline scalar | MRays/s (est.) | Notes |
|---|---:|---:|---:|---|
| **Baseline scalar** (shadows+reflections, adaptive off, depth 5) | 0.5271 | — | 1.75 | reference |
| Shadows OFF | 0.4392 | -16.7% time | 2.1 | interactive-equivalent scalar |
| Reflections OFF | 0.3055 | -42.0% time | 3.02 | no recursive bounce path |
| Shadows OFF + Reflections OFF | 0.2356 | -55.3% time | 3.91 |  |
| Adaptive ON (½ samples) | 0.1943 | -63.1% time | 2.37 | ~half primary rays vs baseline |
| max_depth 3 (was 5) | 0.4834 | -8.3% time | 1.91 | shallower ray_color recursion |
| SIMD packets (vs scalar baseline) | 0.2902 | 81.7% faster | 3.18 | AVX2 packet path; same spp |
| Wavefront (vs scalar baseline) | 0.2371 | 122.4% faster | 3.89 | tiled wavefront path |
| Morton traversal (vs scalar baseline) | 0.3506 | 50.3% faster | 2.63 | Z-curve pixel order |

## How to read **vs baseline**

- **Scalar rows:** *% time* = `100 * (t - t_baseline) / t_baseline` — **negative** means faster than baseline.
- **SIMD / wavefront / Morton:** *% faster* = `100 * (t_baseline / t - 1)` — **positive** means faster than baseline scalar.
- **Adaptive:** uses half the primary samples; MRays/s uses the effective sample count.
