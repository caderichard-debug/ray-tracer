# Feature delta benchmark

- **Scene:** Cornell box
- **Resolution:** 320x180
- **Samples (baseline):** 2
- **Timed reps (median):** 2, warmup: 1
- **OpenMP threads:** 2

| Configuration | Median time (s) | vs baseline scalar | MRays/s (est.) | Notes |
|---|---:|---:|---:|---|
| **Baseline scalar** (shadows+reflections, adaptive off, depth 5) | 0.0634 | — | 1.82 | reference |
| Shadows OFF | 0.0568 | -10.4% time | 2.03 | interactive-equivalent scalar |
| Reflections OFF | 0.0355 | -44.0% time | 3.24 | no recursive bounce path |
| Shadows OFF + Reflections OFF | 0.032 | -49.5% time | 3.6 |  |
| Adaptive ON (½ samples) | 0.02124 | -66.5% time | 2.71 | ~half primary rays vs baseline |
| max_depth 3 (was 5) | 0.05349 | -15.6% time | 2.15 | shallower ray_color recursion |
| SIMD packets (vs scalar baseline) | 0.0344 | 84.3% faster | 3.35 | AVX2 packet path; same spp |
| Wavefront (vs scalar baseline) | 0.03523 | 80.0% faster | 3.27 | tiled wavefront path |
| Morton traversal (vs scalar baseline) | 0.04217 | 50.4% faster | 2.73 | Z-curve pixel order |

## How to read **vs baseline**

- **Scalar rows:** *% time* = `100 * (t - t_baseline) / t_baseline` — **negative** means faster than baseline.
- **SIMD / wavefront / Morton:** *% faster* = `100 * (t_baseline / t - 1)` — **positive** means faster than baseline scalar.
- **Adaptive:** uses half the primary samples; MRays/s uses the effective sample count.
