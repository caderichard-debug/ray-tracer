Features

High-level overview of what ships today. The CPU interactive build is the primary surface; GPU and ASCII are included for completeness.

CPU interactive (SDL2)

  • Quality ladder from small preview resolutions up to 1920×1080-class presets with adjustable samples per pixel.
  • Camera: WASD movement, mouse look when captured, arrow keys for vertical motion.
  • Progressive, adaptive, and wavefront paths for trading latency vs throughput (toggles in the live panel).
  • Analysis modes: cycle through normals, depth, albedo, and composite views for debugging and teaching.
  • Post pipeline: denoise amount control and variance-guided blending on the RGB buffer.
  • Live histogram of luminance (toggle with G) for exposure debugging.
  • Window screenshots capture the render target after the scene draws and before UI overlays, so PNGs match what you see in the viewport.
  • Translucent settings panel composited over the framebuffer for quick tweaks without leaving the scene.
  • Camera hot reload from disk (F5) for rapid layout iteration.

Batch CPU

  • Offline PNG output with anti-aliasing up to 256 samples per pixel.
  • Command-line flags for resolution, depth, Morton ordering, and deterministic runs for testing.

ASCII terminal

  • Cross-platform terminal renderer with animated camera orbit presets—no GUI required.

WIP. OpenGL-backed interactive mode is under active development; expect missing features vs CPU and avoid performance claims until benchmarks are reproducible on main.
